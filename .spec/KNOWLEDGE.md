# KNOWLEDGE - ドメイン知識・調査結果

## 業務・ドメイン知識
- テトリスガイドライン：SRS回転（JLSTZ/I別キックテーブル）、7バッグ、ロックディレイ500ms（移動リセット最大15回）、T-spinは3コーナールール＋フロントコーナーでMini判定、キック5番目(±1,-2)でフル昇格
- T-spin判定は「最後の成功操作が回転」が条件。ハードドロップ距離0ならlastRotateを維持する実装にすることでスピン入れ→ドロップ確定が成立する
- スコア：Single100/Double300/Triple500/Tetris800、TSS800/TSD1200/TST1600相当、B2B×1.5、コンボ50×n×lv、パーフェクトクリアボーナス

## 調査・リサーチ結果
- **T-spin連発の定石調査（2026-07-16、Web徹底調査済み）**：
  - LST積み＝3列目（index 2）をノッチ兼井戸にし、S/L（またはZ/J）で交互に屋根を作ってTSDをループする定石
  - TSDスロットの数学的定義：ノッチ列cに対し h[c±1]=h[c]+1、屋根セル(c±1, h[c]+2)が片側のみ、2行がc-1,c,c+1以外埋まっている
  - Cold Clearボットの核心：①「近い将来入手できるTの数だけ、スロット検出→仮想消去→再検出を繰り返し多段スロット全部に加点」②穴を cavity(-173)/overhang(-34)/covered(-17) に3分類し屋根由来の穴を軽罰 ③wasted_t(-152)でTの無駄遣いを罰する ④単純消去clear1-3はマイナス報酬
  - 詳細レポートは調査エージェント出力参照（TKI開幕・DT砲のテンプレ座標も取得済み）
- Dellacherie評価関数（着地高・消去セル・行/列遷移・穴・井戸）は1手読みで半永久に生存する強さ。ただし「浅く消し続ける」性質でT-spin/テトリスは一切狙わない
- ヘッドレスシミュレーション（esbuildでバンドル→node実行）が評価関数チューニングに極めて有効。400ミノ×戦略別で定量評価した

## 技術的な知見
- Cold Clear流の多段先読み＋overhang軽罰＋wasted_t罰の導入で、T-spin消去が8→10〜17回/400ミノに向上（シミュレーション計測、planMove平均8.5ms/手）
- 対戦実装：ClearInfo→攻撃力テーブル（TSD4/TST6/テトリス4/B2B+1/コンボ/PC+10）→自分の受けと相殺→相手のgarbageQueueへ。せり上がりは「消去しなかったロック後」に適用（上限8行/回）
- 複数盤面描画はBoardView（THREE.Group内包）方式でレンダラを分離。カメラはfitCameraが盤面数に応じて距離を自動調整
- **T-spinスロットの幾何判定はニセ検出だらけ**：コーナー3つ＋2行揃いの「見た目スロット」の9割以上はスピンで入れない。BFS（回転が最後・接地・コーナー3・2行消去）による到達可能性検証を入れて初めてTSDが安定して出る（1回→8〜11回/400ミノ）
- スロット保持ボーナスが高すぎると「スロット博物館」（作ったまま絶対使わない）になる。実行ボーナス >> 保持ボーナスの序列が必須
- 戦略実装は評価関数へのボーナス/ペナルティ注入方式が簡潔：combo=左3列ペナルティ＋フェーズ制、tetris=右端ペナルティ＋準備完了行ボーナス＋4列消し大ボーナス
- Three.js: InstancedMeshのinstanceColorで200セルを1ドローコール描画。UnrealBloomPassは強度0.55/閾値0.72程度が上品（強すぎるとフレームが眩しすぎる）
- カメラは fitCamera()（盤面半径/tan(fov/2)、アスペクト比考慮）で全盤面を常に画面内に収める
- Web Audio重厚化：デチューン2枚サウ＋サブ矩形のリード、3層ベース、キック/スネア/ハット、フィードバックディレイ、小節頭コード刺し、DynamicsCompressorで音圧
- ブラウザの自動再生制限のため、AudioContextはタイトル画面のクリックで初期化する

## 決定事項と理由
- ビルド環境：Vite 6 + TypeScript strict + three ^0.170（@types/threeと同minor固定）
- vite base './'：GitHub Pagesのサブパス（/tetris/）でも相対参照で動く
- 外部アセット不使用（BGM/SFXは合成、フォントのみGoogle Fonts）：著作権クリーン＆軽量
- AI操作もInputBus経由の仮想ボタンにする：画面上のコントローラが人間/AI問わず光る要件を単一経路で実現
- 高重力時（gravitySec<0.18）はAIの滑り込み経路を無効化：実行中の自然落下とのズレによる誤設置を防ぐ
